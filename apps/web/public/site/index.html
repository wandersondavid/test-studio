Como funciona

O fluxo pensado para o dia a dia de quem precisa testar sem travar o time.

O objetivo nao e so executar Playwright. E transformar gravacao, reuso, depuracao e reteste em algo simples o suficiente para ser adotado por produto, operacao e QA.

Recursos principais

Os recursos que realmente importam para colocar o produto em uso agora.

A plataforma foi moldada pelos gargalos mais comuns em automacao interna: seletor fragil, login repetido, fluxo assincrono, necessidade de rerun e debug dificil.

REC

Record from here

Escolha qualquer step ja gravado e reabra o recorder no ponto certo. O produto reexecuta setup, replay parcial e deixa voce continuar sem voltar ao zero.

SET

Login e setup reutilizavel

Use cenarios base e blocos com parametros para reaproveitar autenticacao, navegacao inicial e qualquer trecho repetido do fluxo.

API

waitForApi e retry dinamico

Espere resposta de API, visibilidade, texto ou URL com tentativas e intervalos configuraveis. Ideal para fila, emissao e processamento assincrono.

DBG

Preview por step com logs

Abra a execucao, selecione qualquer etapa e veja screenshot, console e rede para encontrar a falha sem depender apenas da stack trace.

SEL

Seletor mais resiliente

O recorder prioriza data-testid, guarda alternativas de seletor e prepara o terreno para self-healing sem transformar cada ajuste em regravacao.

CLI

Operacao por UI e terminal

Rode por cenario, suite ou tudo de uma vez via interface ou comando. O mesmo produto serve para o time visual e para automacao em fluxo local ou CI.

Em vez de deixar a automacao espalhada entre codigo, planilha e memoria do time, o Test Studio centraliza gravacao, execucao, diagnostico e ownership.

Perguntas que aparecem quando o produto sai do experimento e vai para o uso real.

Sim. O runner foi pensado para abrir localhost da maquina hospedeira, e o recorder usa uma sessao real do Playwright, o que funciona muito melhor para ambiente local e dev interno do que um iframe comum.
